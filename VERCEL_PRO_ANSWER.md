# Vercel Pro $20 使用额度说明

## 💰 每月 $20 使用额度（不是余额）

### 说明
- **这是每月可用的 $20 额度**，用于所有资源（构建、带宽、计算、存储等）
- **不是余额**：这是每月分配给你的使用额度
- **灵活使用**：可以用于任何资源类型

### 如何使用
- **构建时间**：使用 Standard 构建机器（不启用 On-Demand）完全免费，包含在 $20 额度内
- **带宽**：用于 CDN 和 API 请求
- **计算**：用于 Serverless Functions
- **存储**：用于 Edge Config、Blob Storage 等

### 超出后
- 超出 $20 后按实际使用付费
- 建议设置预算警告（$15 和 $25）避免意外高额账单

## 🚀 关于构建机器性能

### ⚠️ 重要：构建机器性能与批量生成文案问题无关

**原因**：
1. **构建机器只影响构建时间**，不影响运行时性能
2. **批量生成文案在 Serverless Functions 中运行**，不受构建机器影响
3. **Enhanced/Turbo 性能**只会在构建时使用，不会影响 API 调用性能

### 建议
- **使用 Standard 构建机器**（不启用 On-Demand Concurrent Builds）
- **完全免费**，包含在 $20 额度内
- **不需要 Enhanced 或 Turbo**，因为与你的问题无关

## 🔧 批量生成文案问题修复

### 问题
1. Gemini 2.5 生成文案不能保存
2. 代码错误无法继续生成批量文案

### 已修复
1. ✅ **改进错误处理**：即使某个批次失败，也会继续下一个批次
2. ✅ **改进保存机制**：即使某些场景词保存失败，也会继续保存其他场景词
3. ✅ **添加错误恢复**：不会因为单个错误而停止整个任务

### 修复内容
- 所有保存操作都添加了 try-catch，确保即使保存失败也能继续
- 所有生成操作都添加了错误处理，确保即使生成失败也能继续下一个批次
- 改进了错误日志，更容易定位问题
