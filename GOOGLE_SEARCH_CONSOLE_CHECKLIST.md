### 步骤 1: 验证网站所有权（必须完成）

**检查方法**：
1. 进入 Google Search Console
2. 查看网站属性状态
3. 确认是否显示"已验证"或"Verified"

**如果未验证**：
- **方法A - HTML文件验证**（如果使用）：
  - 检查 `public/googlec426b8975880cdb3.html` 文件是否存在
  - 确认可以通过 `https://your-domain.com/googlec426b8975880cdb3.html` 访问
  - 访问验证，应该显示验证成功

- **方法B - DNS验证**（推荐）：
  - 查看 Google Search Console 中提供的TXT记录
  - 确认DNS中已添加该TXT记录
  - 等待几分钟后点击"验证"

**验证状态**：
- [ ] ✅ 已验证 / ❌ 未验证

---

### 步骤 2: 提交 Sitemap（必须完成）

**检查方法**：
1. 在 Google Search Console 中
2. 进入 **"站点地图"** 或 **"Sitemaps"**
3. 查看是否已提交 `sitemap.xml`

**如果没有提交**：
1. 点击 **"添加新的站点地图"** 或 **"Add a new sitemap"**
2. 输入：`sitemap.xml`
3. 点击 **"提交"** 或 **"Submit"**

**提交状态**：
- [ ] ✅ 已提交 sitemap.xml
- [ ] ✅ 状态显示"成功"或"Success"

**你的 sitemap URL**：
```
https://your-domain.com/sitemap.xml
```

**如果提交失败，检查**：
- sitemap 是否可以访问：访问 `https://your-domain.com/sitemap.xml`
- sitemap 格式是否正确：应该是 XML 格式
- 确保 sitemap 中包含至少一个 URL

---

### 步骤 3: 手动请求索引主页（重要）

**为什么重要**：
- 虽然提交 sitemap 后 Google 会自动发现页面
- 但手动请求索引可以更快地被抓取

**操作方法**：
1. 在 Google Search Console 顶部搜索框
2. 输入你的主页 URL（例如：`https://your-domain.com/`）
3. 点击 **"测试实际网址"** 或 **"Test Live URL"**
4. 等待检查完成
5. 如果显示 **"可编入索引"** 或 **"URL is on Google"**
   - 点击 **"请求编入索引"** 或 **"Request Indexing"**

**请求状态**：
- [ ] ✅ 已请求索引主页
- [ ] ✅ Google 显示"已请求编入索引"

---

3. 查看 **"未编入索引"** 或 **"Excluded pages"** 是否有问题

## 🔍 常见问题排查

### 问题 1: 验证失败

**可能原因**：
- HTML 文件无法访问
- DNS 记录未正确配置
- 等待时间不够（DNS 更改可能需要几分钟）

**解决方案**：
1. 确认验证文件可以公开访问
2. 检查 DNS 记录是否正确
3. 等待 5-10 分钟后重试

### 问题 2: Sitemap 提交失败

**可能原因**：
- Sitemap URL 不正确
- Sitemap 格式错误
- Sitemap 无法访问

**解决方案**：
1. 直接访问 sitemap URL 确认可以访问
2. 使用 [XML Sitemap Validator](https://www.xml-sitemaps.com/validate-xml-sitemap.html) 验证格式
3. 确保 sitemap 包含至少一个 URL

### 问题 3: 请求索引后仍搜不到

**可能原因**：
- Google 需要时间处理（通常 1-7 天）
- 网站内容质量不高
- 缺少外链

**解决方案**：
- **耐心等待**（新网站通常需要几天到几周）
- **继续创建高质量内容**
- **建立外链**（帮助 Google 发现你的网站）

### 问题 4: 抓取频率很低

**可能原因**：
- 网站是新站
- 没有外链
- 内容更新频率低

**解决方案**：
- **建立外链**（帮助 Google 发现新内容）
- **定期更新内容**
- **保持网站活跃**
