## 📊 两个限制的区别

### 一、「OAuth 用户数上限 100」的真实含义

**显示名称**：New User Authorization Limit（新用户授权上限）

**真实含义**：
- ✅ **每天"第一次使用 Google 登录你这个应用的新增用户"最多 100 个**
- ✅ **只统计「第一次授权」**
- ✅ **老用户重复登录不算**
- ✅ **已经用过一次 Google 登录的用户永远不再占这个名额**

**不是**：
- ❌ 不是 DAU（日活跃用户）
- ❌ 不是 MAU（月活跃用户）
- ❌ 不是你产品能有的总用户数

**举例**：
```
如果你今天有 1 万老用户登录 + 80 个新用户注册
→ 完全没问题 ✅
```

---

### 二、「每日令牌授权率 10,000」的真实含义

**显示名称**：Token Issuance Rate（令牌发放率）

**真实含义**：
- Google 每天最多给你发 **10,000 次 OAuth access token**
- 包括：登录、token refresh 等所有 token 请求

**实际影响**：
- 这个值非常高，对 SaaS 来说几乎不可能撞到
- 一天可以：10,000 次登录 或 10,000 次 token refresh
- **可以直接忽略这条限制**

## 📚 参考信息

### Google OAuth 限制类型

1. **New User Authorization Limit（新用户授权上限）**
   - 默认：100 / 天
   - 只统计第一次授权的新用户
   - 老用户重复登录不计入

2. **Token Issuance Rate（令牌发放率）**
   - 默认：10,000 / 天
   - 包括所有 token 请求（登录、refresh）
   - 对 SaaS 来说几乎不可能撞到

### 申请流程

1. **访问**：Google Cloud Console → OAuth consent screen
2. **点击**："Increase New User Authorization Limit"
3. **填写表单**：
   - Project ID
   - Homepage URL
   - Email address
   - Expected daily new users
   - Reason for increase
4. **等待审核**：3-5 个工作日
