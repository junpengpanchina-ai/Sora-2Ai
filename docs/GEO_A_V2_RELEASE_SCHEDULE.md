# 15万页面 Google 发布节奏表

> **目标**：让 Google 能"吞下去"15万页面，不被识别为 spam，保持健康索引

---

## 🎯 核心原则

**你不是被内容拖死，是会被「一次性发布规模」拖死。**

Google 对程序化站点的真实行为：
- 新域名 / 弱域名
- 大量相似结构
- 一次性放量

👉 **不会处罚，但会**：
- 慢抓
- 少索引
- 延迟评估（sandbox 感）

---

## 📅 四阶段发布节奏

### 阶段 1：信任建立期（Day 1–14）⚠️ 已更新为降速版

**目标**：让 Google 确认内容可读、不是 spam、有真实结构变化

| 指标 | 数值 |
|------|------|
| **每天发布** | **20–40 页**（降速） |
| **总量** | **300–500 页** |
| **累计** | **~500 页** |

**⚠️ 重要变更**：
- 从 30-50 页/天 → 20-40 页/天
- 给 Google 足够时间"理解你在干嘛"
- 不要触发 programmatic spam 的自动限速

**关键动作**：
- ✅ 确保内容质量（使用 GEO-A v2 随机结构）
- ✅ 提交初始 sitemap（≤ 500 URLs）
- ✅ 监控索引率（目标：> 60%）
- ✅ 检查抓取错误（确保无 404/500）

---

### 阶段 2：线性扩展期（Day 15–45）⚠️ 已更新为降速版

**目标**：建立稳定的抓取预算，让 Google 识别为"use-case 型资源站"

| 指标 | 数值 |
|------|------|
| **每天发布** | **20–40 页**（保持降速） |
| **每周上限** | **200–300 页**（不是 1,500） |
| **累计** | **~2,000 页** |

**⚠️ 重要变更**：
- 从 150-300 页/天 → 20-40 页/天
- 从 1,500 页/周 → 200-300 页/周
- 继续给 Google 足够时间"理解你在干嘛"

**关键动作**：
- ✅ 提交分组 sitemap（按行业）
- ✅ **不要每天更新 sitemap index**（每周更新一次）
- ✅ 监控索引率（目标：> 70%）
- ✅ 检查抓取频率（Search Console）

**Sitemap 分组示例**：
```
/sitemap-industry-dental.xml
/sitemap-industry-real-estate.xml
/sitemap-industry-education.xml
/sitemap-industry-healthcare.xml
```

每个 sitemap：
- ≤ 2,000 URLs
- 行业一致
- use case 混排

## 🚨 风险控制

### 如果索引率下降（< 60%）

1. **立即暂停发布** 1–2 天
2. **检查内容质量**：
   - 是否有重复内容？
   - 是否使用了随机结构池？
   - 是否有薄内容（< 500 词）？
3. **检查技术问题**：
   - 服务器响应时间
   - 抓取错误（404/500）
   - robots.txt 配置

### 如果抓取频率下降

1. **降低发布速度**（减少 50%）
2. **检查服务器负载**
3. **优化页面性能**（TTFB < 500ms）

### 如果出现抓取错误

1. **立即修复**（404 → 301 重定向或删除）
2. **更新 sitemap**（移除错误 URL）
3. **监控 24 小时**（确保无新错误）
