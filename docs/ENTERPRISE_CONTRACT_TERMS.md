# Enterprise 合同核心条款（Billing / SLA / Refund）

## 版本定位
不是律师长文，而是**"能被采购和法务快速接受"的商业版条款**  
可直接放进合同正文或 Term Sheet

---

## 1️⃣ 计费与结算（Billing）

### 1.1 计费方式

- 本服务采用 **Credits 预充值制**
- 每生成 1 条成功视频，按对应模型消耗固定 Credits
- **仅对成功生成的视频计费**

### 1.2 Credits 定义

- Credits 为平台内部计费单位
- 不可提现，不可转让
- **企业 Credits 永久有效，不设过期时间**

### 1.3 批量任务结算规则

**批量任务（Batch Job）创建时：**
- 系统会根据任务数量 × 单条成本 **预扣 Credits**

**批量任务完成后：**
- 实际成功视频数量 × 单条成本 = 实际消耗
- **失败视频对应 Credits 将自动退回账户**
- 结算过程全自动完成，无需人工介入

### 1.4 定价

**Pay-as-you-go（按量付费）：**
- Sora-2: $0.49 / video（或等值 Credits）
- Veo-Pro: $5.99 / video（或等值 Credits）
- 最小批量：10 条

**Enterprise Prepaid（企业预付包）：**
- $2,000: 等效 ~380 Veo videos（~12% 折扣）
- $5,000: 等效 ~1,050 Veo videos（~20% 折扣）
- $10,000: 等效 ~2,300 Veo videos（~27% 折扣）
- Credits 永久有效，支持月结发票

## 2️⃣ 失败退款（Refund Policy）

### 2.1 自动退款机制

**若视频生成任务失败（系统错误 / 模型失败）：**
- 对应 Credits **自动退款**
- 退款完成后实时反映在账户余额中
- **无需人工干预，无需工单申请**

### 2.2 不可退款场景

- 客户主动取消任务
- 因客户提供非法、违规或侵权内容导致任务被拒绝
- 因客户系统故障导致无法接收 Webhook 通知（不影响退款，仅影响通知）

### 2.3 对账透明

- 每一次扣费与退款均记录在 **Credits Ledger**
- 客户可通过 Admin Dashboard 或 API 随时查询历史记录
- 支持 CSV / JSON 格式导出，便于财务对账

### 2.4 退款时效

- 退款在任务失败确认后 **实时完成**
- 退款立即回到账户余额，可继续使用
- 所有退款记录可在 Admin Dashboard 实时查看

## 3️⃣ 服务等级协议（SLA）

### 3.1 可用性

- **平台年度可用性目标：99.9%**
- 不包括第三方模型供应商（如 Sora / Veo）不可控中断
- 不包括客户网络问题、API Key 被禁用等客户原因

### 3.2 处理时效（非硬 SLA，企业可选升级）

- Batch Job 通常在合理时间内开始执行
- 支持并发执行，具体并发数以合同约定为准
- **Sora-2**: 95% 任务在 2 分钟内完成
- **Veo-Pro**: 95% 任务在 10 分钟内完成
- 超时任务自动标记失败并退款

### 3.3 幂等保障

- **批量任务幂等保障：100%**
- 所有 API 请求支持 `request_id` 幂等保护
- 相同 `request_id` 不会重复创建 batch，不会重复扣费

### 3.4 失败退款保障

- **失败任务退款：100% 自动**
- 无需工单，无需人工干预
- 系统级退款机制，保证退款及时到账

### 3.5 SLA 违约处理（可选条款）

- 若因平台原因导致连续性大规模失败：
  - 双方可协商补偿 Credits
  - 不涉及现金赔偿
- 若平台可用性低于 99.9%：
  - 按合同约定提供 Credits 补偿
  - 具体补偿比例以合同为准

## 4️⃣ API 与安全

### 4.1 API Key

- 每个企业客户可创建多个 API Key
- 支持：
  - 启用 / 禁用
  - 调用限流（rate_limit_per_min）
  - 使用审计（usage logs）
  - IP 白名单（可选）

### 4.2 安全与风控

- 所有请求均具备幂等保护（`request_id`）
- 系统具备异常调用监测与自动限流能力
- 支持 Webhook HMAC-SHA256 签名验证
- 所有 API 调用记录在 `enterprise_api_usage`，可审计

### 4.3 数据安全

- 所有视频生成记录存储在客户账户下
- 支持数据导出（CSV / JSON）
- 支持数据留存策略（30/90/180 天，可配置）
- 符合 GDPR 数据导出要求

## 5️⃣ 法律与责任边界（关键一条）

### 5.1 内容责任

- **平台不对生成内容的合法性承担最终责任**
- 客户需确保其输入内容及使用场景符合当地法律法规
- 客户需确保生成内容不侵犯第三方知识产权

### 5.2 服务中断

- 因第三方模型供应商（Sora / Veo）服务中断导致的失败：
  - 任务自动标记失败并退款
  - 平台不承担额外责任
- 因平台系统故障导致的失败：
  - 按 SLA 条款处理
  - 提供 Credits 补偿（如适用）

### 5.3 数据备份

- 平台提供数据备份机制
- 所有 Credits 变动、Batch 记录、Task 记录均有完整备份
- 系统恢复后，所有未完成 Batch 会继续执行

---

## 6️⃣ 合同终止与退款

### 6.1 合同终止

- 任何一方可提前 30 天书面通知终止合同
- 合同终止后，API Key 将被禁用
- 未使用的 Credits 可继续使用（永久有效）

### 6.2 终止后 Credits 处理

- **未使用的 Credits 永久有效**
- 客户可继续使用 Credits 完成已创建的 Batch
- 不支持 Credits 提现或转让

---

## 7️⃣ 争议解决

### 7.1 对账争议

- 如客户对账单有异议，可在 30 天内提出
- 双方基于 `credit_ledger` 记录进行对账
- 如确属平台错误，平台将立即修正并补偿

### 7.2 技术争议

- 如客户对系统功能有异议，可通过技术支持渠道反馈
- 平台将在合理时间内响应并解决
- 如确属系统 bug，平台将修复并补偿（如适用）

## 关键卖点总结

### 对客户最有价值的条款

1. **Credits 永久有效** → 降低采购风险
2. **失败自动退款** → 成本可控
3. **完整审计** → 财务安心
4. **幂等保障** → 技术可靠

### 对平台最有价值的条款

1. **内容责任边界** → 降低法律风险
2. **第三方服务中断免责** → 降低运营风险
3. **Credits 不可提现** → 保证现金流
