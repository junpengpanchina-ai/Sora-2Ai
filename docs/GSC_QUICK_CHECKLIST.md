## ⚡ 每周 5 分钟检查

### 1️⃣ 打开 GSC → Pages（2 分钟）

| 指标 | 记录值 | 健康阈值 | 状态 |
|------|--------|---------|------|
| **已编入索引** | _____ | - | - |
| **未编入索引** | _____ | - | - |
| **编入比例** | _____% | ≥ 70% | ⬜ |

**计算公式**：`编入比例 = 已编入索引 / (已编入索引 + 未编入索引) × 100%`

| **被阻止** | _____ | ❌ = 0 | 需修复 |

---

### 3️⃣ 打开 GSC → Sitemaps（1 分钟）

| 检查项 | 状态 | 说明 |
|--------|------|------|
| `/sitemap.xml` 状态 | ⬜ | 应为"成功" |
| 最后读取时间 | ⬜ | 应在 24 小时内 |
| 已发现网页数 | ⬜ | 应与 Pages 数据一致 |

## 📊 状态判断

### 🟢 健康期（继续）

**条件**：
- ✅ 编入比例 ≥ 70%
- ✅ 软 404 < 3%
- ✅ 被阻止 = 0

**行动**：
- ✅ 维持当前节奏
- ✅ 持续监控

---

### 🟡 限速期（需注意）

**条件**：
- ⚠️ 编入比例 50-70%
- ⚠️ 软 404 3-5%
- ⚠️ 被阻止 = 0

**行动**：
- 🔻 发布量减半
- 🔻 修复软 404
- 🔻 增强内容差异化

---

### 🔴 风险期（需处理）

**条件**：
- 🚨 编入比例 < 50%
- 🚨 软 404 > 5%
- 🚨 被阻止 > 0

**行动**：
- ⛔ 立刻停止发布 7 天
- ⛔ 修复所有软 404
- ⛔ 检查 robots.txt / noindex

---

## 🔍 快速诊断 SQL

### 检查软 404 风险页面

```sql
SELECT id, slug, title, LENGTH(content) as content_length
FROM use_cases 
WHERE is_published = true 
  AND in_sitemap = true
  AND (content IS NULL OR LENGTH(content) < 300)
ORDER BY content_length ASC
LIMIT 50;
```

### 检查被阻止的页面

```sql
SELECT id, slug, title, noindex, in_sitemap
FROM use_cases 
WHERE is_published = true 
  AND (noindex = true OR in_sitemap = false)
LIMIT 50;
```
