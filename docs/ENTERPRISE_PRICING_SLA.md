## 技术保障

### 资金安全

- **预扣机制**：批量任务创建时预扣整单 credits
- **结算机制**：成功数 × 单价 = 实际扣费
- **自动退款**：失败任务自动退款，无需人工干预
- **账本审计**：所有资金变动记录在 `credit_ledger`，可追溯

### 幂等性保障

- **request_id 幂等**：相同 `request_id` 不会重复创建 batch
- **RPC 幂等**：`freeze_credits_for_batch` 和 `finalize_batch_credits` 支持幂等调用
- **Worker CAS**：使用数据库 CAS 更新，避免重复执行

### 可观测性

- **Batch 状态追踪**：queued → processing → completed/partial/failed
- **Task 级别追踪**：每个任务的生成状态、错误信息、失败类型
- **Webhook 投递追踪**：投递次数、HTTP 状态码、失败原因
- **账本记录**：所有资金变动（预扣、结算、退款）完整记录

## 企业级能力

### 批量生成

- 单次 batch 支持 10 - 10,000 条任务
- 并发控制：可配置并发数，避免下游压力
- 部分成功：支持 partial 状态（部分成功、部分失败）

### 访问控制

- 播放 / 下载 / embed 统一访问决策
- 下载次数 / 并发 / 速率可控
- 全量访问审计（合规级）

### Webhook 通知

- HMAC-SHA256 签名验证
- 超时控制（默认 5s）
- 指数退避重试（最多 3 次）
- 失败不影响结算
