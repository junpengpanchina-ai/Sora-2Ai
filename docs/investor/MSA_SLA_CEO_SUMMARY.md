# MSA + SLA CEO 摘要版（2 页）

> **用途**：CEO / 董事会 / 投资人快速阅读
> **目的**：看"我们承担什么、不承担什么"

---

## Page 1｜合同的"责任边界图"

### 一句话总览

```
Sora2 sells safe scaling infrastructure — not guaranteed SEO outcomes.
```

---

### 1️⃣ 我们对 Enterprise 客户承诺什么（✔）

> 我们承诺的是"可控性"，不是"结果"

| 承诺项 | 说明 |
|--------|------|
| ✔ Enterprise Batch API | 稳定可用，99.5% SLA |
| ✔ Index Health Dashboard | 实时可观测 |
| ✔ SEO 风险自动阻断 | Scaling Gate 自动生效 |
| ✔ Sitemap 完整性校验 | 防 silent failure |
| ✔ 风险隔离 | SEO 问题不影响业务交付 |

**关键词**：Visibility · Control · Isolation

---

### 2️⃣ 我们明确不承诺什么（✖）

> 以下全部写死在 MSA 里

| 不承诺项 | 原因 |
|----------|------|
| ✖ Google 收录 | 第三方决定 |
| ✖ 搜索排名 | 第三方决定 |
| ✖ 流量增长 | 多因素影响 |
| ✖ SEO KPI | 不可控 |
| ✖ 算法变更后果 | 不可控 |

**这是防无限责任的核心**

---

### 3️⃣ 自动阻断权（非常重要）

```
Sora2 有权在检测到 SEO 风险时自动冻结扩容
```

| 触发条件 | 后果 |
|----------|------|
| Index Rate < 40% | 扩容冻结 |
| Tier1 空 chunk | 扩容冻结 |
| 连续无增长 | 扩容冻结 |

**关键点**：
- 不构成违约
- 不影响已购服务 / Batch 交付
- 属于"风险保护行为"

**这条是我们"敢卖 Enterprise"的底气**

---

### 4️⃣ 费用与定价的本质

| 定价方式 | 说明 |
|----------|------|
| 固定费 / 年付优先 | 不按 usage 计费 |
| Credits = 使用权限 | 不是结果承诺 |
| 不做 usage-based SEO 定价 | 因为 SEO 风险是系统级的 |

---

## Page 2｜SLA 的"承诺清单"

### 一句话总览

```
SLA only covers what we can control.
```

---

### 5️⃣ 我们 SLA 承诺的指标（✔）

> 只承诺系统层

| 指标 | 目标 |
|------|------|
| API / Dashboard 可用性 | 99.5% |
| Batch 处理完成率 | 99% |
| Dashboard 数据更新 | 日级 |
| SEO 风险告警延迟 | < 1 小时 |

**SLA = 系统稳定性，不是 SEO 效果**

---

### 6️⃣ SLA 明确排除项（✖）

| 排除项 | 原因 |
|--------|------|
| ✖ Index Rate 数值 | 第三方决定 |
| ✖ 实际收录数量 | 第三方决定 |
| ✖ 流量 / 收入变化 | 多因素影响 |
| ✖ SEO 成功或失败 | 不可控 |

---

### 7️⃣ 责任上限（防灾条款）

| 条款 | 说明 |
|------|------|
| 赔偿上限 | 最近 12 个月费用 |
| 排除间接损失 | 流量 / 收入 / 商誉 |
| SEO 结果不在 SLA 范围 | 明确写死 |

---

### 8️⃣ CEO 级总结句

```
We don't promise outcomes we can't control.
We promise we'll never scale irresponsibly.
```

## 投资人 / 董事会 FAQ

### Q: 这份合同最大的风险在哪？

```
A: 最大风险是客户误解"SEO Infrastructure"等于"SEO 保证"。
   我们通过 MSA 明确排除、销售话术反复强调来规避。
```

### Q: 如果客户 SEO 失败，会告我们吗？

```
A: 合同明确排除 SEO 结果责任，且责任上限 = 12 个月费用。
   我们不承诺不可控结果，只承诺系统能力。
```

### Q: 为什么不按 usage 定价？

```
A: 因为 SEO 风险是系统级的，不是单页级的。
   按 usage 定价会激励过度扩容，与我们的风控目标冲突。
```
