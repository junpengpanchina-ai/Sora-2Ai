# Investor Q&A - Hardball Version

> **用途**：投资人尽调、Board Meeting、战略讨论
> **原则**：不回避尖锐问题，展示成熟思考

---

## Business Model Questions

### Q1: "你不保证 SEO 结果，那客户为什么要买？"

**Answer (慢一点说)**:

```
因为 Enterprise 客户最怕的不是"没增长"，
而是一次失控导致整个域名被伤害。

我们卖的是：不把客户域名当实验品。
```

**Key point**: We sell risk management, not outcomes.

---

### Q2: "如果 Index Rate 一直不好，那客户不是一直用不了？"

**Answer**:

```
那说明当前策略不适合扩。

我们宁愿客户暂时扩不了，
也不会让他们承担长期 SEO 损害。
```

**Why this is mature**: We prioritize customer health over our revenue.

---

### Q3: "不按 usage 计费，会不会限制收入上限？"

**Answer**:

```
相反，这是我们能做 Enterprise 的前提。

Usage-based 定价会激励我们放任扩容，
但我们的系统设计目标正好相反：
在不安全时阻止客户。
```

**The insight**: Our pricing aligns with our product design.

---

### Q4: "那客户会不会觉得你们'限制他赚钱'？"

**Answer**:

```
Enterprise 客户更怕的是"赚钱后再被清算"。

我们是在帮他们控制最昂贵的失败成本。
```

**Framing**: We're not limiting upside, we're protecting downside.

---

## Competitive Questions

### Q5: "这种 Infra 能形成壁垒吗？"

**Answer**:

```
能，因为它不是功能，是组织能力。

它要求技术、产品、定价、合同四件事完全一致。

大多数团队做不到这一点。
```

**Why this is strong**: True moats are organizational, not technical.

---

### Q6: "如果竞争对手抄你的 Dashboard 怎么办？"

**Answer**:

```
Dashboard 是表象，背后是：
- 数据库约束
- CI 阻断器
- 定价哲学
- 合同条款

抄界面很容易，抄整套系统设计很难。
```

---

### Q7: "便宜的竞争对手会不会抢走客户？"

**Answer**:

```
会抢走追求便宜的客户。

但追求便宜的客户不是我们的目标市场。

我们的客户是：已经在规模边缘、怕出事的团队。
```

---

## Risk Questions

### Q8: "如果 Google 彻底改变规则，你怎么办？"

**Answer**:

```
那我们依然做同一件事：
提前暴露风险，阻止不可控扩容。

我们不赌规则，我们管理不确定性。
```

**Key point**: Our value is in risk management, not prediction.

---

### Q9: "客户会不会因为 SEO 失败告你们？"

**Answer**:

```
合同明确排除 SEO 结果责任。
责任上限 = 12 个月费用。

我们不承诺不可控结果，只承诺系统能力。
```

---

### Q10: "如果整个 AI SEO 赛道被 Google 打击怎么办？"

**Answer**:

```
那我们会是最后倒下的。

因为我们的系统设计本来就是：
在 Google 不配合时保护客户。

如果赛道真的死了，至少我们的客户损失最小。
```

---

## Strategy Questions

### Q11: "你们适合所有客户吗？"

**Answer**:

```
不适合。

如果只是试验、小规模，Starter 就够了。

Enterprise 是给已经在规模边缘的团队。
```

**Why this is good**: Shows market discipline.

---

### Q12: "你们为什么不做 usage-based 定价？"

**Answer**:

```
If we charged per page, we'd be incentivized to let customers over-generate.

Our model only works if we stop them when it's unsafe.
```

---

### Q13: "你这家公司本质上卖的是什么？"

**Answer (停 1 秒再说)**:

```
We sell the ability to scale without regret.
```

---

## Unit Economics Questions

### Q14: "你们的毛利是多少？"

**Answer**:

```
Enterprise Core: ~64%
Enterprise Scale: ~70%+

随着规模增长，边际成本下降，毛利会提升。
```

---

### Q15: "LTV 是多少？"

**Answer**:

```
Enterprise Core: $60k-$90k (2-3 年)
Enterprise Scale: $180k+ (2+ 年)

高留存是因为 SEO Infra 深度嵌入客户工作流。
```

---

### Q16: "CAC 是多少？"

**Answer**:

```
Enterprise 获客主要靠：
1. 内容营销（技术博客、复盘文章）
2. 口碑转介
3. 销售直接触达

CAC/LTV 比例健康，具体数字取决于渠道。
```

---

## The Ultimate Question

### Q17: "为什么是你们？"

**Answer**:

```
因为我们踩过这个坑。

我们自己经历过 sitemap 指向空文件、GSC 显示成功但收录为零的事故。

然后我们把这个教训变成了产品。

大多数团队还在假装 SEO 是确定性的。
我们是少数承认它不确定、并为此设计系统的团队。
```

## Summary: The Mature Founder Response Pattern

1. **Don't dodge** - Acknowledge the concern
2. **Reframe** - Show it from a different angle
3. **Evidence** - Point to concrete design choices
4. **Confidence** - End with a clear position
