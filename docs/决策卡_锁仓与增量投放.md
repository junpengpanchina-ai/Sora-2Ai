# 决策卡：锁仓与增量投放

> **最强一句话**（对任何“想扩”冲动的统一答案）：  
> **Lockdown 未解除且未 GREEN：系统层面不允许扩容，证据链也不足以证明稳定，因此不做增量投放。**

**数据流**：你用 **表格**（图表、Coverage 导出等）**上传达标反馈数据** → **Admin** 根据这些数据判断（GREEN / LOCKDOWN、是否允许扩展等）。决策以 **Admin 判断** 为准。

3. **Owner 明确解除 Lockdown / 允许扩展**

## 二、关键数据缺口（必须补齐，否则 Admin 无法判断“稳定”）

> 判断“无法证明稳定”不是保守，是**证据链缺口**导致的必然结论。这些数据填进**表格**、上传后，**Admin** 才能据此判断。

最少需要补齐 **3 类数据**（都用 **过去 7 天滚动窗口**），填入表格并上传：

### 1. Crawl / Discovered / Indexed 的增量曲线（表格须填报，供 Admin 判断）

- daily discovered
- daily crawled
- daily indexed
- **index rate = indexed / crawled**（体系内核心指标）

### 2. Coverage 的异常项

- Valid with warnings
- Excluded（按原因分组）：
  - Duplicate
  - Crawled - currently not indexed
  - Soft 404
  - Blocked by robots
  - Alternate page with proper canonical
  - 等

### 3. Sitemap tier 的健康度

- tier1-0 是否持续被抓
- 新增 sitemap 是否引起 crawl 分配波动（含此前 off-by-one 教训）

## 三、LOCKDOWN 期间允许做的“安全动作”（不触发扩容）

这些动作**不改变页面总量、不改变索引目标**，但提升后续 GREEN 概率。

### 1. 修复类（允许）

- 纠正 canonical / hreflang / meta robots 的一致性
- 修复 sitemap 指针、分页、lastmod（**不新增 URL**）
- 处理明显 404 / 500 / 重定向链路（减少 crawl 浪费）

### 2. 质量类（允许）

- 对**已索引 / 已被抓**的页面做轻量内容补强（**不加新页**）
- 内链整理：从首页 / 场景 hub 指向已索引页（属于“分配权重”，不是“扩容”）

### 3. 观测类（强烈建议）

- **每天固定导出同一份 GSC Coverage**（或 API 拉取同口径），做 **7 日滚动对比**
- 记录：Indexed 总数、Excluded 总数、主要 excluded reason 的变化 → **填入表格、上传，供 Admin 判断**

> **注意**：Trends 仅做决策参考。LOCKDOWN 期间可用 Trends 生成「微调关键词建议清单」，但**不要产新页**。

## 四、一旦转 GREEN：增量投放 SOP（小步快跑，防二次失控）

**目标**：在不破坏 Index Gate 的前提下，逐步扩大“可索引内容池”。

### Step 0：解锁条件（建议写死在 Owner checklist）

**同时满足**（你填进表格、上传后，**Admin 据此判断**）：

- Indexed / Crawled / Discovered 等有连续 **≥3 天** 可读数（非 "-"）
- **index rate** 连续 **≥3 天** 不下降（可设阈值，如 **≥70%**）
- Coverage 异常项无爆发（Excluded 任一类不出现连续上升）

### Step 1：第一档增量（最小增量）

- **只放行一个 bucket**（例如：一个国家、一个 use case、或一个 scene 的增量）
- 增量规模：**+5% ~ +10%**（相对当前可索引量），**不要翻倍**

### Step 2：观察窗（至少 72 小时）

看 3 件事：

- crawled 是否激增但 indexed 不动（典型“消化不过来”）
- excluded reason 是否出现新峰值
- tier1-0 抓取是否被挤压

### Step 3：第二档增量（通过则继续）

- 同样 **+5% ~ +10%**
- 仍只扩**一个** bucket，避免多变量无法归因

### Step 4：回撤信号 → 立刻回 LOCKDOWN

**触发条件**（满足即停扩、回锁仓）：

- index rate **连续 2 天** 下降
- excluded 某类原因 **连续 2 天** 上升（尤防 **Crawled - currently not indexed**）
- crawl 被新 sitemap 抢走，导致核心页面抓取减少

---

## 五、阈值速查

| 项目 | 阈值 / 条件 |
|------|-------------|
| 解锁 | 表格有 ≥3 天可读 · index rate 不降 ≥3 天 · Excluded 无连续升 → Admin 可放行 |
| index rate 建议 | ≥70%（可依实际调） |
| 单档增量 | +5% ~ +10%，单 bucket |
| 观察窗 | ≥72 小时 |
| 回撤 | index rate 连续 2 天降 **或** excluded 某类连续 2 天升 **或** tier1 抓取被挤压 |
