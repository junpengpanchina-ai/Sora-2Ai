## Section 2: How It Works

**标题**: How It Works

**流程**:

1. **Submit a batch** (API or UI)
   - Upload prompts, configure models, set parameters
   - Single request for 10-10,000 videos

2. **Credits are pre-checked and frozen**
   - Balance validation before execution
   - Full batch cost frozen upfront

3. **Videos are generated asynchronously**
   - Sora-2 or Veo-Pro models
   - Concurrent execution with configurable limits

4. **Failed tasks auto-refund**
   - Network errors, model failures, timeouts
   - Automatic refund to wallet

5. **Webhook + admin dashboard for tracking**
   - Real-time status updates
   - Complete audit trail

**一句话**:
> No manual reconciliation. No surprises.

## Section 6: Technical Details

**标题**: Production-Grade Architecture

**核心能力**:

- **Batch Execution**: 10-10,000 videos per job
- **Financial Safety**: Pre-check → Freeze → Settle → Refund
- **Idempotency**: 100% guarantee via `request_id`
- **Concurrency Control**: Configurable worker limits
- **Webhook Delivery**: HMAC-signed, retry-backed
- **Admin Dashboard**: Full reconciliation view

**技术栈**:
- Backend: Next.js + Supabase (PostgreSQL)
- Models: Sora-2, Veo-Pro (via Grsai API)
- Worker: Internal worker with configurable concurrency
